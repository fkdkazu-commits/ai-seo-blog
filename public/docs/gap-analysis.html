8フェーズの実装仕様と追加開発オプション

1Step

ファクト

情報収集

一部対応

ベース対応

Playwrightが指定キーワードでGoogle検索を自動実行し、上位競合記事のH2/H3見出し構造を抽出する。検索意図・カバレッジギャップ・差別化余地を分析し、次ステップの執筆プロンプトへ渡す入力データを自動生成する。

対象国内SERP（日本語検索結果）のみ
出力次ステップのプロンプト入力として内部処理（ファイル出力なし）

追加仕様

  • 海外SERP（英語圏の論点・観点）を収集する
  • SNS・Yahoo知恵袋から読者の悩み・感情を収集する
  • 医療・健康テーマでは信頼情報源（病院・大学・公的機関）を優先して収集する
  • 成果物（リサーチメモ・上位記事分析シート・悩み感情リスト・信頼情報源リスト）を人間が読める形で出力する
追加費用 2万円

考察

ユーザー理解と記事方針の設計

未実装

ベース対応

対応なし。リサーチ後そのまま執筆に進む。

  • 検索意図・潜在ニーズ・差別化ポイントの仮説を提示する
  • 根拠と紐付けたユーザー像を整理する
  • 「読者理解シート」「記事方針メモ」を出力する
追加費用 1万円
3Step

設計

コンテンツブリーフ作成

未実装

対応なし。Step 1のリサーチ出力がそのまま執筆プロンプトに渡される。

4Step

判断

人間確認・承認

未実装　最大差分

ベース対応

対応なし。全処理が完全自動のため、リサーチから執筆・公開まで人間が介入するポイントがない。

追加仕様

  • パイプラインを一時停止して人間の承認を待つ仕組みを実装する
  • 承認通知（Slack等）・承認UI・再開トリガーを実装する
  • 専門性・ブランド方針・事業目的の観点でのチェックポイントを設ける
  • 表現リスク・引用元妥当性の確認フラグを付与する
  • 「承認済みコンテンツブリーフ」「執筆時の注意事項」を出力する
追加費用 3万円
5Step

制作

執筆

一部対応

ベース対応

Playwright × Claude.aiによる5段階パイプライン（①リサーチ → ②構成 → ③ファクトチェック → ④品質編集 → ⑤内部リンク）を順次実行し、MDX形式の記事全文を自動生成する。

品質E-E-A-T準拠・禁止表現リスト・数字整合性チェック・FAQ・まとめを組み込んだプロンプト群で制御
出力MDXファイル（フロントマター・本文・内部リンク含む）をGitHubへ自動コミット

追加仕様

  • 見出し単位（H2ごと）に逐次生成する
  • 各見出しで「結論→理由→具体例→注意点→根拠→つなぎ」を個別制御する
  • YMYL分野では信頼情報源を優先し、SNS・知恵袋を根拠として使用しないルールを適用する
追加費用 2万円
6Step

改善

編集・校正・ファクトチェック

一部対応

ベース対応

Step5の品質編集フェーズが、AI典型表現・責任回避表現・冗長表現を自動検出して修正する。数字とリストの整合性（例：見出しの「7ステップ」と実際の項目数のズレ）を検出・修正する。MDX構文エラー（見出し記法・テーブル・箇条書き）も自動修正する。

チェック禁止表現パターン9カテゴリ（読者否定・主語不明・AI前置き・冗長 等）
出力修正済み記事全文（チェックリスト・修正メモの別途出力はなし）

追加仕様

  • 冗長表現・論理の飛躍・重複・検索意図ズレを構造的に指摘し、改善案を提示する
  • 「要確認箇所リスト」「修正メモ」を成果物として出力する
  • 数値・医療情報・法規制等の人間確認が必要な箇所をフラグ付きで明示する
追加費用 1万円
7Step

実装

マークアップ・入稿

一部対応

ベース対応

Unsplash APIでOG画像を自動取得・設定し、priority・changefreq付きの sitemap.xml を自動生成する。mainブランチへのpushでGitHub Actionsが起動し、Astroビルド → GitHub Pagesデプロイまでを無人実行する。

CMSGitHub Pages（静的サイト）固定。外部CMSへの入稿は非対応
確認公開前の人間レビューポイントなし。pushと同時に自動公開

追加仕様

  • WordPress等の外部CMSへの入稿に対応する
  • 画像権利・容量・alt設定を自動チェックする
  • 計測タグ設定を補助する
  • 人間が最終公開判断を行うフローを設ける
  • 「公開前チェックリスト」を出力する
追加費用 3万円
8Step

検証

公開後検証・リライト

一部対応

ベース対応

Google Search Console APIで過去28日間のクリック数・表示回数・平均順位・CTRを自動取得し管理ダッシュボードに表示する。3段階の自動検出でリライト候補を優先順位付けして提示する。

判定1順位ドロップ：11〜30位かつ表示50回以上（Quick Win狙い）
判定2CTR不足：1〜10位かつ期待CTRの70%未満（First Page Sage 2026基準）
判定3低インプレ：公開90日超かつ表示50回未満（SurferSEO基準）

追加仕様

  • 想定検索意図と実際の流入キーワードのズレを自動検出する
  • CTR低下タイトルを自動検出する
  • CV（コンバージョン）連携分析を実施する
  • リライト指示書を自動生成する
  • 「公開後レポート」「リライト指示書」を出力する
追加費用 2万円
